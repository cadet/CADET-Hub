# Contributing

CADET-Hub is an open-source project and we warmly welcome contributions from the community.

You are encouraged to open issues or submit patches (preferably as pull requests) at any time. For larger planned contributions or architectural changes, we recommend discussing them first through a GitHub issue or the [CADET forum](https://forum.cadet-web.de/).

For detailed guidance on development workflows, installation, dependencies, code style, testing, and model development within the CADET ecosystem, please refer to the [CADET Developer Guide](https://cadet.github.io/master/developer_guide/index.html) and follow Research Software Engineering (RSE) best practices as outlined in the [RSE guide](https://rse-guidelines.readthedocs.io/en/latest).

---

# Issues & Pull Requests

We use **GitHub issues** and **pull requests** to manage development and collaboration.

Please follow the provided [issue and PR templates](.github/ISSUE_TEMPLATE/) and adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).

## Issues vs. Pull Requests

### Issues

Use **GitHub issues** for:

* Reporting bugs
* Suggesting new features
* Discussing technical questions, ideas, or design decisions

Please choose the appropriate issue template and provide clear descriptions and reproduction steps where applicable.

### Pull Requests

Submit **pull requests** once your changes are implemented and tested.

To help maintain a manageable review process:

* Keep PRs focused on **one issue or feature**
* Reference the related issue(s) in the PR description
* Ensure tests pass and documentation is updated where relevant

---

# Branch Policy

To keep development organized, we follow a simple branch naming convention:

* **Feature branches**
  `feature/your-feature-name`

* **Bug fixes**
  `fix/short-description`

* **Hotfixes**
  `hotfix/issue-description`

* **Documentation updates**
  `docs/update-description`
